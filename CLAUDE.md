# Verso App — Route Generation Web App

## What This Is
FastAPI web application that wraps the POC 8 costmap routing engine with a Leaflet map UI. Click anywhere on the map to generate a scored running route in under 1 second. Includes data layer visualization as blurred heatmap overlays.

**GitHub:** https://github.com/verso-run/verso-app
**Port:** 8090
**Deployment:** Docker (Railway-ready)

## Structure
```
app.py                  ← FastAPI server: SSE route generation, layer API, geocoding
config.py               ← Composite weights, paths, constants (shared with POC 8)
templates/index.html    ← Leaflet frontend: click-to-route, layer toggle, route cards
graph/helpers.py        ← edge_id, normalize_tag, save/load graph
router/astar.py         ← A* with turn penalty, haversine heuristic
router/astar_rx.py      ← Direction-expanded graph (scipy CSR Dijkstra, 8x speedup)
router/loops.py         ← Loop generation: lollipop, out-and-back, full loop
router/loops_fast.py    ← Optimized loop generation
router/export.py        ← GeoJSON + GPX export
Dockerfile              ← Python 3.12-slim
railway.json            ← Railway deployment config (healthcheck on /api/layers)
data/                   ← Scored graph pickle + layer JSONs (gitignored, large)
```

## Running Locally
```bash
# Requires scored graph from POC 8 compose step
# Copy data/graphs/sf_scored.pickle and data/layers/*.json from poc8-costmap

python app.py
# Open http://localhost:8090
```

## Running with Docker
```bash
docker build -t verso-app .
docker run -p 8090:8090 verso-app
```

## How It Works
1. On startup, loads the pre-built scored graph (`sf_scored.pickle`) into memory
2. User clicks a point on the map (or enters an address for geocoding)
3. Server generates loop routes via A* with turn penalty on the scored graph
4. Routes streamed back via SSE, rendered as Leaflet polylines
5. Data layers (way tags, landuse, elevation, etc.) available as toggleable heatmap overlays

## Key Features
- **Click-to-route:** <1s route generation from any point on the map
- **Heatmap layers:** Data layers rendered as blurred, zoom-adaptive heatmap washes with rainbow gradient
- **SSE streaming:** Route generation progress streamed to the frontend in real time
- **Multiple loop strategies:** Lollipop, out-and-back, full loop — ranked by composite score

## Data Dependencies
The app requires pre-built data from `route-valhalla-poc/poc8-costmap`:
- `data/graphs/sf_scored.pickle` — scored graph with running_cost per edge
- `data/layers/*.json` — layer scores for heatmap visualization

These files are large (100MB+) and gitignored. Generate them by running the POC 8 build pipeline.
